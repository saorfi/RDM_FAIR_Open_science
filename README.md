## Standard operating procedures (SOP)
[10 simple rules on how to write a standard operating procedure](https://journals.plos.org/ploscompbiol/article?id=10.1371/journal.pcbi.1008095)
"Measures to ensure reuse and reproducibility need to
include the entire research life cycle, from the experimental design to the generation of data,
quality control, statistical analysis, interpretation, and validation of the results."
"Despite their importance, the need for the use of standards in life science research has emerged
as crucial for the quality and reproducibility of research findings only in recent years [9]."
1. Knowing when to write an SOP
    * repeated in reproducible way
    * Often templates exist somewhere (optimal shared server / cloud with internal SOPs)
    * Involve:
      * Technicians
      * group leader
      * quality manager
      * data manger
2. Write the introduction: describe the purpose( the why)
    * define research environment
    * specification
    * simplicity
3. Set up document structure
    * Cover page
      * administrative information about institution
      * title identifying activity /procedure
      * SOP ID, number with versioning
      * page number / total number of pages
      * Date of issue / versioning
      * safety instructions
      * name of individual who prepared and approved the SOP
      * the name of the reviwers and date of review
      * description of the purpose and field of application
      * name and function of author
      * name and function of approver
        + Typical abbreviations for SOP category  
          - F- Fundamental
          - M / METH - analytical method
          - P - procedure to carry out a special investigation
          - PROT - protocol
          - ORG - organizational document
          - WS - worksheet
          - A/ APP - apparatus
          - SOP; P / PROJ   - procedure to carry out special investigation(project)
          - PERS - to describe personnel matters
          - RF - registration form
      * Recommendation: using digital object identifier DOI / for identification of SOP in accordance with [ISO 26324:2012](https://www.iso.org/standard/43506.html)
      * Footing note ( administrativ info, title SOP, page number, date of approcal and /or version number)- Using source control systems like GitHub
      * Using date format specified in [ISO 8601](https://www.iso.org/iso-8601-date-and-time-format.html)
    * Sequence of steps or tasks (metadata) for given procedure
    * list of references and definitions
4. Fill in the content
    * Aim of SOP
    * SOP may contain multiple SOPS
    * work instruction via stepwise process strictly.
    * Balance the level of detail, avoid unnecessary specification and alternatives. If alternatives, dicatet which application
    * Consistent terminology, layout, media and method
    * break the process into sections
    * break the sections into specific steps
    * number the steps or bullet point for clarity
    * describe each task in detail ( including timeframes and tools required)

5. References and Definitions: Specify tools required for the task
  * Tools: modeling and simulation tools, data repos and compiler construction tools
  * references: instructions, other standards, research papers, graphical material, photographs , other SOPs.
  * Provide definitions and abbreviations

6. Set up responsibilities and nominate reviewers and approvers
    * reviwer: scientific knowledge and experise as well as experience in the field
    * Open discussion on aspects of the SOP
    * Each change an own revision
    * Agreement on timeframe to monitor and align SOP

7. Test with a colleague : Perform training
    * Check for clarity and understandability
    * Execute test run
    * all appropriated tatt trained with and familiarized with new SOP

8. Review and approve:
    * All agree on document -> send to quality manager
    * Major changes- Terminate, label and archive SOP
    * Stored at central location

9. Update document: Specify validation and periodic review Date
    * regular update
    * adjust version numbers
    * Ensure interoperability of used formats to enable export of the data into other systems ( when using ELN)

10. Publish:
   * secured team server or cloud
   * or making available to a broad user community e.g. Zenodo, SEEK, OpenAIRE, FAIRsharing ,...
